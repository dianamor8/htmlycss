Sass

Sass es un lenguaje de hoja de estilo que se compila en CSS. Le permite usar variables, reglas anidadas, mixins, funciones y más, todo con una sintaxis totalmente compatible con CSS.

Sitio oficial: https://sass-lang.com/

Características

  1. Compatible con CSS

    Sass es completamente compatible con todas las versiones de CSS, para que pueda usar sin problemas cualquier biblioteca CSS disponible.

  2. Definición de variables

    Sass ofrece soporte para la creación de variables, éstas permiten 4 tipos de datos los cuales son: números, String o cadenas de caracteres, colores en hexadecimal o por nombre (en ingles) y valores de tipo booleano.

  3. Partials

    Un partial es un archivo Sass nombrado al principio con un subrayado inicial. Puedes asignarle un nombre como _variables.scss y escribir en él la declaración y asignación de variables en Sass de el proyecto.

    Es importante destacar que el guión bajo le permite a Sass saber que el archivo es solo un archivo partial y que este sera llamado con la directiva @import desde el archivo principal .scss o .sass. Esto ayuda a ordenar, separar o modularizar a nivel de archivo tu código Sass.

  4. Importación

    La opción o directiva de importación @import permite dividir junto con los archivos partials el CSS en partes pequeñas y fáciles de mantener. Solo hay un inconveniente que se detalla en la documentación oficial y es que cada vez que se utiliza un @import en CSS se crea otra solicitud HTTP.

  5. Nesting (Anidamiento)

    Sass te permitirá anidar tus selectores CSS de una manera que siga una jerarquía visual de tipo HTML. Pero hay que tener en cuenta que las reglas excesivamente anidadas resultarán en CSS difícil de mantener y se considera una mala práctica. Así que no abusar de muchos anidamientos y sobre-anidamientos de tus selectores.

  6. Mixins

    Un mixins es parecida a una función (semejante a la de un lenguaje de programación) que podrá ser usada en distintos selectores, lo que permitirá escribir valores o atributos CSS y reutilizarlos adecuadamente. Esto es uno de los aportes que mencionaba que Sass ayudaba a hacer bloques de código de forma modular.

  7. Herencia

    Esta es otra de las caracteristicas más utiles de Sass. Haciendo uso de la palabra reservada @extend te permite compartir un conjunto de propiedades CSS de un selector a otro.

  8. Operators

    Sass ofrece un puñado de operadores matemáticos estándar como son: Adición (+), Sustracción (–), Multiplicación (*), División (/) y Modulo (%).

¿Cómo empezar?

Instalación independiente

Puede instalar Sass en Windows, Mac o Linux descargando el paquete para su sistema operativo desde GitHub y agregándolo a su PATH. Eso es todo, no hay dependencias externas y nada más que deba instalar.

Puede instalar Sass globalmente para proporcionar accesoal ejecutable usando:

npm install -g sass

También puede agregarlo al proyecto para proporcionar un ejecutable y una biblioteca usando:

npm install node-sass

Escriba éste código en su archivo package.json 


{
  "scripts": {
    "scss": "node-sass — watch scss -o css",
  }
}

Cree una carpeta scss y cree un archivo styles.scss

Cree una carpeta css

Ejecute el script

npm run scss

Escriba el código dentro de styles.scss 

              
$test: red;
body{
  background-color: $test;
}

Finalmente, compile nuevamente
