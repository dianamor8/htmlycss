Module Bundler

Es un software que se encarga de analizar el árbol de relaciones y dependencias existentes entre los distintos módulos de un programa y de empaquetarlos en un archivo único, que puede ser consumido por el browser.

Por lo general, un module bundler comienza con un archivo de entrada, y desde allí agrupa todo el código necesario para ese archivo de entrada. Hay dos etapas principales de un paquete: Dependency resolution (Resolución de dependencia) y Packing (Embalaje)

Beneficios de usar module bundler

  • El navegador no es del todo compatible con el sistema de módulos, pero por otro lado los módulos nos proveen un mecanismo básico de encapsulación y un bloque de construcción fundamental para organizar nuestro código.
  • Ayuda a administrar la relación de dependencia de su código. Es decir que cargará los modulos dependencia en orden.
  • Evita el conflicto de nombres entre archivos.
  • Los módulos nos proveen un mecanismo básico de encapsulación y un bloque de construcción fundamental para organizar nuestro código.
  • Permiten definir loaders, middlewares y plugins para aplicar transformaciones al código que le pasamos como input.
